# SQL 쿼리 최적화 프로젝트 - 아이디어 정의

## 🎯 프로젝트 컨셉
**"SQL Mastery 30: 데이터 분석가 면접을 위한 실무형 SQL 챌린지 (최적화 Before/After 포함)"**

---

## 💡 핵심 기능

### 핵심 기능 1: **30개 실무 시나리오 기반 SQL 문제**
- 이커머스, SaaS, 핀테크 등 도메인별 실전 문제
- "7일 Rolling MAU 계산", "코호트별 재구매율", "Churn Rate" 등
- 각 문제마다 비즈니스 맥락 + 테이블 스키마 + 예상 결과 제공

### 핵심 기능 2: **최적화 Before/After 비교**
- 비효율적 쿼리 → 개선된 쿼리 2-step 제시
- Execution Time, Scanned Data 크기 비교
- "왜 빨라졌는가" 기술적 설명 (인덱스, CTE, Window Function 등)

### 핵심 기능 3: **면접 설명 스크립트**
- "이 쿼리를 5분 안에 설명하세요" 연습용 답변 템플릿
- "면접관이 물어볼 만한 질문 3가지" 미리 준비
- STAR 기법 적용 (Situation → Task → Action → Result)

## 🚨 기존 솔루션의 문제점

### 문제 1: **추상적이고 비현실적인 문제**
- "employees 테이블에서 부서별 평균 급여" 같은 교과서적 문제
- 실무에서는 "이번 달 신규 가입자의 D7 리텐션이 낮은 이유" 같은 질문

### 문제 2: **최적화 경험 불가능**
- 정답만 맞으면 끝, 성능 개선 경험 없음
- "이 쿼리가 왜 느린가?" 학습 기회 부재

### 문제 3: **면접 대비 부족**
- 쿼리는 짜지만 "왜 이렇게 짰는가" 설명 못함
- "다른 방법도 있나요?" 질문에 답변 불가
