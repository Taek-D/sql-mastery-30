CHANGELOG.md를 업데이트합니다.

변경 내용: $ARGUMENTS (예: "Day 31 문제 추가", "case11 최적화 사례 추가")

## 업데이트 규칙

1. 최신 항목을 파일 상단에 추가 (날짜 기준 내림차순)
2. 같은 날짜에 여러 변경이 있으면 기존 날짜 블록에 추가
3. 날짜 형식: `## YYYY-MM-DD`

## 필수 포함 항목

```markdown
## {날짜} — {제목}

### 완료된 사항
- **{카테고리}**: {구체적 변경 내용}
  - 세부 사항 1
  - 세부 사항 2

### 생성/수정된 파일
| 파일 | 작업 | 설명 |
|------|------|------|
| {파일 경로} | 신규/수정/삭제 | {설명} |
```

## 카테고리 분류

| 카테고리 | 설명 | 예시 |
|----------|------|------|
| 문제 추가 | 새 SQL 문제 | Day 31: 피벗 테이블 |
| 최적화 사례 | 새 최적화 Before/After | case11: Index Merge |
| 면접 가이드 | 새 면접 설명 가이드 | guide06: 실시간 대시보드 |
| 버그 수정 | 기존 콘텐츠 오류 수정 | day15.md 쿼리 오류 수정 |
| 설정 변경 | Claude Code / CI 설정 | Hooks 강화, 새 커맨드 추가 |
| 웹앱 개선 | 웹 인터페이스 변경 | 모바일 반응형 개선 |

## 주의사항

- Git 커밋 해시는 커밋 이후에만 기록 (커밋 전이면 생략)
- 변경 파일 목록은 실제 변경된 파일만 기재
- CHANGELOG.md 자체 수정은 파일 목록에 포함하지 않음
